# ADR-0005: 유지형·챌린지 스코프 복제 음성 — 기간제 보존·스코프 고정·추출 차단

- Status: accepted
- Date: 2026-07-23
- Owner: architect
- DECISIONS.md entry: #22

## Context
PRD v1.1은 두 가지 새로운 복제 음성 사용처를 요구한다:
1. **2인 소셜 챌린지(AC-041/044)** — 사용자1이 자기 목소리를 복제해 지인(사용자2)에게 **비동기**(무료 링크 3일) 딥보이스 챌린지를 보낸다. 사용자2가 나중에 체험해야 하므로 음성을 세션 종료 즉시 지울 수 없다.
2. **조건부 목소리 재사용(AC-046)** — 메신저→보이스 에스컬레이션에서 "이미 녹음해 둔 목소리를 재사용"하려면 재사용 가능한 보관 음성이 필요하다.

그러나 기존 **ADR-0003(생성물 즉시 폐기)**는 "세션 종료 시 클론 음성을 Storage와 ElevenLabs 양쪽에서 삭제한다"를 불변식으로 세워 두었다. 위 두 요구는 이 즉시 폐기와 정면으로 충돌한다. 동시에 AC-041은 유출·추출·무단 재사용을 강하게 금지하고, AC-050은 등급별 안전장치 약화를 금지한다. 즉시 폐기 불변식을 **약화하지 않으면서** 보존 요구를 수용할 구조가 필요하다.

## Decision
복제 음성을 **두 개의 명확히 구분된 수명 클래스**로 나눈다.

- **(A) 세션 클론(기존, ADR-0003 그대로):** 솔로 보이스 훈련 세션의 클론. **세션 종료 즉시 폐기.** 기본값·무변경.
- **(B) 유지형/챌린지 스코프 클론(신규, 본 ADR):** 사용자가 **명시적으로** "보관" 또는 "챌린지 생성"을 택했을 때만 생성되는 별도 저장소. **즉시 폐기 대신 기간제 보존**하되, 다음 보증을 강제한다.
  - **스코프 고정:** 챌린지 voiceId는 그 챌린지 문서 컨텍스트 + Functions 자격증명 발급(유효 토큰 + 동의 완료 + 미만료)을 통해서만 해석된다. 다른 챌린지·용도로 재사용되지 않는다(챌린지 1:1). 보관함(`users/{uid}/voices`) 음성은 본인만 접근.
  - **추출 차단:** raw 오디오 바이트·ElevenLabs voiceId를 반환하는 콜러블·다운로드 경로가 사용자1·사용자2 어디에도 없다. 재생은 서버 매개 통화 자격증명(기존 `createRealtimeCall` 패턴)으로만.
  - **기간제 보존:** `retentionDeleteAt`(챌린지 기본 30일, 조정 7~90일) 도달 시 자동 삭제 + 언제든 수동 삭제. 삭제는 **ADR-0003의 폐기 기계(ElevenLabs voice DELETE + Storage 삭제 + `deletionLogs` 기록)를 재사용**한다.

| Option | Pros | Cons |
|---|---|---|
| (B) 별도 수명 클래스 + 기간제·스코프 고정·추출 차단 ✅ | ADR-0003 불변식을 약화 아닌 **범위 한정**으로 보존. 비동기 챌린지(AC-044) 성립. 추출·재사용 차단(AC-041) | 보관 음성이 보존기간 동안 존재 → 폐기 기계·삭제 스캐너 추가 필요 |
| 세션 클론을 그냥 오래 남긴다(ADR-0003 완화) | 구현 단순 | AC-041 유출 리스크·ADR-0003 취지 훼손. **채택 불가** |
| 챌린지도 즉시 폐기(비동기 포기) | ADR-0003 무변경 | AC-044 비동기 챌린지(3일 링크) 불성립. **요구 위반** |

## Consequences
- Positive: ADR-0003(세션 즉시 폐기)이 **기본값으로 그대로 유효**하고, 보존은 사용자가 명시적으로 택한 opt-in 경로에서만 발생한다. 추출 차단·스코프 고정·기간제 삭제가 AC-041을 충족한다. 폐기 기계 재사용으로 감사 로그(`deletionLogs`)가 일관된다.
- Negative / accepted trade-offs: 보존기간(기본 30일) 동안 복제 음성이 존재하므로 즉시 폐기보다 노출 창이 길다 → 스코프 고정 + 추출 차단 + 수동 삭제 + 신고 시 즉시 비활성화(§14.5)로 완화. `retentionDeleteAt` 자동 삭제 스캐너(스케줄 함수) 신설 필요.
- Follow-ups required: T36(챌린지 생성·스코프·토큰) 구현 시 자격증명 발급 게이트(토큰+동의+미만료) 강제. `retentionDeleteAt` 스캔 스케줄 함수. `deletionLogs`에 옵셔널 `challengeId`. 보관함(`users/{uid}/voices`) 전체 UI는 P-8(이번 범위 밖)이나 스키마는 T30/T36 비차단용으로 정의. **AC-050 검증(Architecture.md §14.6): tier는 보존기간·개수·만료 등 용량 축에만 영향, 스코프/추출/동의 게이트는 등급 무관 동일.**

- **통화 = 화면 전환 없는 phase 전이**: 수신→오프닝→실시간 역할극은 UX-014 한 화면 안의 phase일 뿐 별개 라우트 이동이 아니다(D-11·D-12). 라우트 구현 방식(단일 라우트 vs 공유 레이아웃)은 architect 판단(UX-014 Architect Handoff·OQ-U7).

- **메신저피싱 분기 연결됨(v1.7, D-22 — D-17 스텁 해소)**: UX-015의 (B) 메신저피싱은 이제 UX-024(메신저 시나리오 선택)→UX-022(카카오/문자 스킨 셸)로 연결된다. 스미싱 링크는 UX-023(인앱 가짜 랜딩)으로만 이동하고, 에스컬레이션 가능 시나리오는 UX-025(목소리 선택)를 거쳐 채팅 중 전이 트리거 시 통화 셸 UX-014로 이음새 없이 이어진다(UF-007). 스킨·표면은 프레젠테이션 전용이라 안전 셸은 표면과 무관하게 동일하다(§13.4).
- **에스컬레이션 = 채널 전이(하나의 세션·하나의 리포트)**: 메신저 단계와 보이스 단계는 별개 세션이 아니라 같은 세션의 채널 전이이며, 종료 시 두 채널 취약 시점을 함께 다루는 단일 리포트(UX-008)·리플레이 해설(UX-018)로 인계된다(AC-035/037, §13.1).

## Interaction Patterns
재사용 패턴. 구현자는 화면별로 재발명하지 말고 이 정의를 따른다.

- **P-1 클론 생성 대기 로딩**: 단계형 진행 표시("목소리 분석 중 → 클론 생성 중 → 준비 완료") + 예상 시간 문구 + 계속 움직이는 진행 애니메이션으로 "멈춘 인상" 제거. 지연이 길어도 최소 상태 문구가 주기적으로 갱신되어야 함. 타임아웃 시 재시도 버튼 + **데모 폴백(사전 준비 클론/녹화본)** 경로 노출(OQ-U3). 감소 모션 설정 존중, 진행 상태를 스크린리더용 텍스트로도 안내.
- **P-2 세션 중 상시 종료**: 통화(UX-014 전 phase, [P1]UX-009)에 "훈련 종료" 컨트롤을 스크롤과 무관하게 고정 노출. 누르면 **확인 다이얼로그 없이 즉시** 세션을 종료하고 UX-007로 이동(AC-006, 어르신 저부하 — D-2). 수신 phase에서는 "거절"이 종료를 겸한다. 종료 후 첫 포커스는 안심 고지 문구.
- **P-3 합성물 표식 노출**: 모든 합성 음성/이미지에 "AI 훈련용 합성" 표식을 **① 화면 상시 라벨(점멸 금지·고정) + ② 오디오 프리롤 안내 문구**로 이중 노출(AC-022). 이미지에는 화면 라벨 + 대체 텍스트. 표식은 색 배지 단독이 아니라 텍스트 포함.
- **P-4 응답 대기·재시도**: AI 응답/합성/리포트 대기 중 스피너 또는 타이핑 인디케이터 표시, 전송 버튼 비활성으로 중복 요청 방지. 실패/타임아웃 시 인라인 오류 + "재시도" 버튼. 핵심 루프 비차단 요소(P1 이미지 등)는 실패 시 조용히 생략.
- **P-5 입력 검증 타이밍**: 게이트(동의·본인확인·필수 선택)는 미충족 시 다음 버튼을 **비활성**으로 사전 차단(제출 후 오류 대신 사전 안내). 오류 메시지는 해당 컨트롤 근처에 아이콘+텍스트로 표시.
- **P-6 권한 거부(마이크)**: 마이크 권한 거부 시 UX-002를 "권한 필요" 안내 상태로 전환하고 재요청 방법을 텍스트로 안내. 녹음 버튼은 잠금.
- **P-7 오프라인**: 네트워크 끊김 시 진행 버튼 비활성 + "연결을 확인해 주세요" 배너. 복구 시 재시도 가능. (하루 스코프상 자동 재연결 큐잉은 구현하지 않음.)
- **P-8 디스컬레이션 톤**: 종료·리포트 화면 문구는 완화 톤·안심 우선("훈련이었습니다", "안전합니다"), "면역됨" 류 과신 표현 금지, "개선 영역" 프레임(PRD Risk).
- **P-9 확인 다이얼로그 사용 원칙**: 되돌릴 수 없는 파괴적 동작에만 확인을 둔다. 세션 종료는 되돌릴 필요가 적고 즉시성이 중요하므로 확인을 두지 않는다(P-2·D-2).
- **P-10 통화 셸 연속성(Call Shell continuity)** *(v1.4 신설, D-11·D-12)*: UX-014의 수신→오프닝→실시간 역할극은 **하나의 고정 통화 셸** 안에서 phase 전이로만 처리한다. 통화 셸 = ① 상단 AI 합성 표식(고정) + 통화 경과 타이머, ② 중앙 발신자 아바타·라벨(callerLabel) + 현재 대사 자막, ③ 하단 "훈련 종료"(상시). phase가 바뀌어도 이 셸은 자리를 지켜, full-page navigation·레이아웃 점프·"채팅창으로 넘어가는" 인상을 만들지 않는다. 오프닝 대사와 사기범 실시간 응답은 **같은 대화 로그·같은 자막 영역**에 누적된다(오프닝 = 사기범 첫 턴).
- **P-11 오프닝→실시간 이음새 없는 전환(seamless handoff)** *(v1.4 신설, D-12)*: 오프닝 마지막 대사의 오디오 재생 종료(onEnded) 시 **별도 확인·"계속" 버튼 없이** 곧바로 실시간 청취("듣고 있어요…")로 넘어간다. 오디오가 없거나 합성 실패로 재생할 게 없으면 즉시 청취로 전환(침묵 실패 금지). 중간에 "통화 종료/completed" 상태를 두지 않는다 — 통화는 사용자가 "훈련 종료"를 누르거나 한도에 도달할 때까지 끊기지 않는다.
- **P-12 드릴다운 단계 내비게이션(step drill-down)** *(v1.5 신설, D-16)*: 시나리오 선택의 3단계(UX-015 유형 → UX-016 방식 → UX-017 시나리오)는 다음 규칙을 공통으로 따른다.
  - **뒤로가기**: 각 단계에 화면 좌상단 "뒤로" 컨트롤을 두어 직전 선택 단계로 복귀한다(1단계 UX-015의 "뒤로"는 UF-001 직전 화면/홈으로). 뒤로 시 앞서 고른 선택은 유지(재입력 강요 금지). 브라우저 뒤로가기와도 일치시킨다(각 단계가 히스토리 엔트리 — 구현 방식은 architect, OQ-U7류 라우트 판단과 무관하게 스펙상 "뒤로=직전 단계").
  - **단계 표시(현재 위치)**: 각 화면 상단에 지금까지의 선택 경로를 텍스트 브레드크럼으로 표시한다(예: 1단계 "① 유형", 2단계 "보이스피싱 › ② 방식", 3단계 "보이스피싱 › 내 목소리 복제 › ③ 시나리오"). 색 단독이 아니라 텍스트로 단계를 알린다(접근성). 진행 점(dot) 표시는 텍스트 라벨을 보조할 때만 사용.
  - **전환 방향성**: 다음 단계로 진행은 "앞으로"(우→좌 슬라이드 또는 콘텐츠 교체), 뒤로가기는 그 역방향으로 애니메이션해 "좁혀 들어간다/되돌아온다"는 공간감을 준다. **감소 모션(prefers-reduced-motion) 설정을 존중**해 모션을 끄면 즉시 교체로 폴백. 애니메이션은 몰입 보조일 뿐 필수 아님(무애니메이션에서도 흐름·기능 동일).
  - **미구현 분기 처리**: UX-015의 메신저피싱처럼 목적지가 아직 없는 분기는 조용히 막지 말고 "준비 중" 안내로 명시(침묵 실패 금지, P-4/P-5와 정합).
- **P-13 리플레이 해설 스텝 내비게이션(replay walkthrough)** *(v1.6 신설, D-18)*: UX-018에서 대화를 시간순으로 되짚을 때, (a) 순차 스크롤과 (b) "다음 신호로/이전 신호로" 점프를 모두 제공한다. 각 신호 지점의 주석은 "N초 시점: 이 말이 ○○ 신호입니다 / 이렇게 대응했어야 합니다" 형식으로 대화 항목 옆에 인라인 표시하고, 신호 강조는 색만이 아니라 아이콘+텍스트로 이중 표기한다. 톤은 "개선 영역" 완화 프레임(P-8)을 따르고 "면역됨" 류 과신 표현을 쓰지 않는다. 신규 신호 주석은 aria-live로 스크린리더에 알린다. 신호가 없는 대화는 "뚜렷한 위험 신호가 없었습니다"를 명시(침묵 금지).
- **P-14 링크 공유(앱이 대신 전송하지 않음)** *(v1.6 신설, D-20)*: 2인 챌린지 링크(UX-019)는 **앱이 카톡/문자로 대신 보내지 않는다** — 사용자1이 직접 붙여 넣게 한다. UI는 (1) OS 공유 시트(`navigator.share`)가 가능하면 "공유하기" 버튼, (2) 불가/미지원이면 링크 텍스트 + "복사" 버튼으로 폴백(침묵 실패 금지). 링크는 스크린리더가 읽을 수 있는 텍스트로도 노출하고, 복사 완료는 색 아닌 텍스트("복사됨")로 안내. 발급된 링크에는 복제 음성 파일이 포함되지 않는다(토큰만, AC-041).
- **P-15 무동의 차단 게이팅(consent-first gate)** *(v1.6 신설, D-19)*: 2인 사용자2(UX-021)는 **명시적 탭 동의 전에는 어떤 복제 음성도 재생되지 않으며, 무동의로 진행하는 버튼·경로 자체가 화면에 존재하지 않는다**(AC-040). 동의 버튼 외의 유일한 대안은 "신고" 또는 "나가기"뿐이다(체험으로 우회하는 경로 없음). 사후 정체 공개(UX-007 재사용)는 이 사전 동의를 대체하지 못한다(AC-040 명시). 이 게이트는 등급·티어와 무관하게 항상 강제된다(AC-050).
- **P-16 메신저 표면 스킨 전환(presentation-only skin)** *(v1.7 신설, D-22)*: 문자형(UX-022)은 UA 자동 감지로 iOS/삼성/기본 스킨을 적용하고(§13.5 규칙표), 상단에 **수동 전환 토글**을 상시 제공한다(AC-031). 감지 실패·데스크톱은 기본 스킨으로 폴백하며 "기기를 자동 인식하지 못해 기본 화면으로 표시합니다. 직접 바꿀 수 있어요"류 안내를 1줄 노출(침묵 실패 금지). **스킨 전환은 외형만 바꾸고 대화 콘텐츠·안전 문구·"AI 훈련용 모의" 표식·면책 배너·"훈련 종료"를 절대 바꾸지 않는다**(스킨=프레젠테이션 전용, 어떤 안전 판정도 게이팅하지 않음, §13.4). 스크린리더 순서·라벨은 스킨과 무관하게 동일.
- **P-17 스미싱 링크 무해화(harmless smishing link)** *(v1.7 신설, D-23)*: 메신저 대화의 링크는 **`displayText`(모의 표기)만 보이고 실 URL이 없다**(스키마에 URL 필드 부재). 탭하면 외부로 이동하지 않고 `fakeLandingId`가 가리키는 인앱 가짜 랜딩(UX-023)으로만 전환된다. 가짜 랜딩의 입력값은 **서버로 전송되지 않고**(콜러블 부재) "입력됨" 가짜 피드백만 준 뒤 채팅으로 복귀. 링크·랜딩 모두 실 브랜드·기관명 없이 무해화하고 "AI 훈련용 모의" 표식을 노출(AC-032/045). 스크린리더는 링크를 "모의 링크"로 안내(실제 링크 오인 방지).
- **P-18 에스컬레이션 전이 연출(seamless channel escalation)** *(v1.7 신설, D-26)*: 메신저→보이스 전이는 세 경로(① 역할극 LLM sentinel 신호를 서버가 감지 → `escalation` 플래그, ② 메신저 max-turn 폴백 자동 전이, ③ 사용자가 "전화로 확인" 명시 버튼 탭)로 시작되며, **UX는 항상 서버가 준 `escalation:{toChannel:"voice"}` 플래그만 보고 연출한다**(자유텍스트 분류 안 함, sentinel 토큰은 서버가 제거해 화면에 절대 노출되지 않음, §13.2). 연출은 "사기범이 전화를 거는" 수신 화면으로 채팅을 전환해 **기존 통화 셸 UX-014의 incoming phase로 이음새 없이** 인계한다(중간 확인·별도 라우트 점프 최소화, P-11 정합). 통화용 voiceId는 UX-025에서 이미 확보돼 녹음으로 끊기지 않는다(D-25). 사전 고지·동의(이미 완료)·상시 종료·합성 표식·종료 고지는 UX-014 그대로(AC-036, 재설계 없음). 전이 실패 시 메신저 종료로 폴백해 단일 리포트를 보장(AC-035, 침묵 실패 금지). **연출 로직은 방향 무관하게 재사용 가능한 구조로 설계**하되 화면은 메신저→보이스만 구현(AC-039).
- **P-19 상시 면책 고지 배너(persistent disclaimer banner)** *(v1.7 신설, D-24)*: 카카오톡형 표면(UX-022)에는 "카카오톡 실제 서비스와 무관한 훈련용 재현입니다" 면책 고지를 **화면 상단 고정 배너로 상시 노출**한다 — **영구 비활성화 불가, 스크롤해도 사라지지 않음, 재진입 시에도 유지**(AC-047/§13.7, UI 상시 요건이며 데이터 필드 아님). 색 아닌 텍스트 포함·점멸 금지. 이 배너는 스킨 전환·대화 진행과 무관하게 자리를 지킨다.
